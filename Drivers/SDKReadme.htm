Software improvements and new functionality added to Andor SDK 2.95.30002.0

 

New Features

1.      Windows 8 Support

2.      USB iStar now supports the option to change the gate width for each frame in a kinetic series.

3.      Added support for cameras that can store sensor specific acquisition configuration settings on the camera to provide backward compatibility with older software versions

4.      Updated Windows USB camera driver to version 1.2.6 giving better authentication support on Windows 7.

5.      The accuracy of the temperature reported by DU885 systems has been improved

6.      Added version information to the atmechelle.dll library

7.      GetCapabilities SDK function now includes the option to test whether a system supports the pre-scan feature.

8.      Support for configuring gate settings on a DG535 through GPIB

9.      GetAcquisitionProgress will now show the accumulation number incremented when in Accumulate mode and cosmic ray filter on.

10.  SetFKVShiftSpeed although deprecated can now be used with non-Classic systems

11.  Disabled fast kinetics for InGaAs

12.  Added GetRelativeImageTimes function to retrieve meta data information for a range of frames

13.  Support for iDus 416 camera

14.  Adding Extended Isolated Crop Mode functionality that allows a cropped image to be acquired from anywhere on the sensor for iXon Ultra.

 

Bug Fixes

1.      Fixes to prevent gating when USB iStar is not acquiring

2.      Fixed issue on Windows 7 which meant the number of images per DMA was fixed at 1

3.      Fixed a 1 row error in the position of a Fast kinetics acquisition.

4.      Fix for reduced frame rate when horizontally binning in crop mode.

5.      Fix for images saved using the SavesAs functions storing an incorrect value for the Pre Amp Gain

6.      Fix for External start trigger mode causing the acquisition to continuously wait and ignore triggers on iXon cameras at 3MHz readout

7.      Removed an arc artefact from the first frame on a DU920 BR-DD

8.      Fix to resolve an accumulated Odd/Even effect on iDus 401-BV cameras

9.      Fixed crash seen on initialising OptAcquire with 64-bit SDK

10.  Fixed iDus target temperature being different from the one that was set

11.  Shamrock device drivers are now signed and so will not generate a security warning during installation.
